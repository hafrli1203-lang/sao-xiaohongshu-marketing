# 설계 문서 (bot_created_rule STEP 1)

**프로젝트:** 잘파는 연구소 - 샤오홍슈 체험단 마케팅 홍보 사이트
**버전:** v1.0
**작성일:** 2026-03-16

---

## 목적

샤오홍슈(小红书) 체험단 마케팅 서비스를 한국 사업주(맛집/카페/뷰티/여행 등)에게 홍보하고, 고객 문의 리드를 수집하는 B2B 랜딩페이지.

## 트리거 조건

- **랜딩페이지 (/):** 사용자가 URL 방문 (수동)
- **문의폼 제출 (/inquiry):** 사용자가 폼 작성 후 제출 버튼 클릭 (이벤트)
- **카카오톡 문의:** 사용자가 카카오톡 QR 스캔 또는 ID 검색 (외부 이벤트)

## 실패 복구

| 실패 상황 | 복구 방법 |
|-----------|----------|
| 폼 제출 API 오류 (500) | 에러 메시지 표시 + "카카오톡으로 문의해주세요" 대안 안내 |
| SMTP 환경변수 미설정 | API Route에서 503 응답 + "이메일 서비스가 설정되지 않았습니다" 메시지 반환 |
| 클라이언트 validation 실패 | 필드별 에러 메시지 표시, 제출 버튼 비활성화 |
| 이미지 로드 실패 | Next.js Image 컴포넌트의 기본 fallback 사용 |
| API 타임아웃 | 클라이언트에서 10초 타임아웃 + 재시도 유도 메시지 |

## 상태 경계

- **독립 사이트:** 외부 시스템과 공유 상태 없음
- **이메일 발송만:** Nodemailer → Gmail SMTP (단방향, 상태 저장 없음)
- **카카오톡:** 외부 링크/QR만 제공 (API 연동 없음)
- **데이터베이스 없음:** 폼 데이터는 이메일로만 전달, 서버에 저장하지 않음
- **타 모듈 충돌:** 없음 (단독 프로젝트)

## 아웃풋 정의

| 아웃풋 | 형태 | 전달 경로 |
|--------|------|----------|
| 랜딩페이지 | HTML 렌더링 (9개 섹션) | 브라우저 → `/` |
| 문의 페이지 | HTML 렌더링 (폼 + 카카오톡) | 브라우저 → `/inquiry` |
| 폼 제출 결과 | JSON `{ success: boolean, message: string }` | API → `/api/inquiry` |
| 문의 이메일 | 텍스트 이메일 (업체명/담당자/연락처/업종/문의내용) | SMTP → hafrli1203@gmail.com |
